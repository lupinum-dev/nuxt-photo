---
title: Use a custom lightbox globally
description: Set one default lightbox component for Photo, PhotoAlbum, PhotoGroup, and PhotoCarousel across the app.
---

Use this guide when most of your app should share the same custom lightbox.

This path keeps per-instance `:lightbox` overrides available, but removes the need to pass the same component into every album, group, carousel, or photo.

## 1. Create your lightbox component

Start with a wrapper around the recipe `<Lightbox>`:

```vue [components/MyLightbox.vue]
<script setup lang="ts">
import { Lightbox } from '@nuxt-photo/recipes'
</script>

<template>
  <Lightbox>
    <template #caption="{ photo }">
      <div class="my-caption">
        <h2 v-if="photo?.caption">{{ photo.caption }}</h2>
        <p v-if="photo?.description">{{ photo.description }}</p>
      </div>
    </template>
  </Lightbox>
</template>
```

If that is not enough, you can swap it later for a full primitive-based lightbox.

## 2. Provide it once

Register the component through `LightboxComponentKey`.

```ts [plugins/nuxt-photo-lightbox.ts]
import { LightboxComponentKey } from '@nuxt-photo/vue'
import MyLightbox from '~/components/MyLightbox.vue'

export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.provide(LightboxComponentKey, MyLightbox)
})
```

Now the default recipe lightbox path uses `MyLightbox` across the app.

## What it affects

The global component is used when these components open their default lightbox:

- `<Photo>` with `lightbox`
- `<PhotoAlbum>` with `lightbox` enabled
- `<PhotoGroup>` with `lightbox` enabled
- `<PhotoCarousel>` with `lightbox` enabled

It does **not** affect components where:

- `:lightbox="false"` disables the lightbox
- `:lightbox="SomeOtherComponent"` sets a per-instance override

Per-instance `:lightbox` wins over the global default.

## When to use this

Choose the global override when:

- the whole app should share one branded lightbox
- you want one place to change caption/actions/counter behavior
- repeated `:lightbox="MyLightbox"` props would add noise

Use [Customize the built-in lightbox](/docs/guides/customize-built-in-lightbox) when only one gallery or one section needs the custom UI.
