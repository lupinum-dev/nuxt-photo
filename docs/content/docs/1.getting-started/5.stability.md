---
title: Stability Contract
description: What is stable, what is advanced, and what is internal.
---

Nuxt Photo is still pre-1.0, but the public contract is intentionally split:

- Root imports from `@nuxt-photo/core`, `@nuxt-photo/vue`, `@nuxt-photo/recipes`, and `@nuxt-photo/nuxt` are the stable surface.
- `@nuxt-photo/vue/extend` is public, but lower-stability. It exists for custom lightboxes, integrations, and advanced composition.
- Anything undocumented is internal, even if it happens to be exported today.

Practical rule:

- Use recipe components for the happy path.
- Use `@nuxt-photo/vue` primitives and composables when you need custom UI.
- Use `@nuxt-photo/vue/extend` only when the documented public surface is not enough.

If you build on `/extend`, expect more churn across breaking releases than you would from the root entries.
