---
title: useContainerWidth
description: Track an element's width with ResizeObserver, optional SSR seeding, and breakpoint snapping.
---

`useContainerWidth()` is a low-level Vue helper for responsive layouts. It watches one container element, returns its width as a ref, and can snap that width down to declared breakpoints so layout work stays stable across resize noise.

```ts
import { useContainerWidth } from '@nuxt-photo/vue'
```

## Signature

```ts
function useContainerWidth(
  containerRef: Ref<HTMLElement | null>,
  options?: {
    defaultContainerWidth?: number
    breakpoints?: readonly number[]
  },
): {
  containerWidth: Ref<number>
}
```

## Parameters

### `containerRef`

Ref for the element you want to observe.

### `options.defaultContainerWidth`

Optional SSR seed width. Use it when you want a server render to start from a known width before `ResizeObserver` runs on the client.

### `options.breakpoints`

Optional list of breakpoint widths. When provided, the composable snaps the observed width down to the largest breakpoint less than or equal to the real width.

## Return value

### `containerWidth`

A Vue `Ref<number>` that updates after mount as the observed element changes width.

## Example

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { useContainerWidth } from '@nuxt-photo/vue'

const containerRef = ref<HTMLElement | null>(null)
const { containerWidth } = useContainerWidth(containerRef, {
  defaultContainerWidth: 1200,
  breakpoints: [375, 768, 1200],
})
</script>

<template>
  <div ref="containerRef">Width: {{ containerWidth }}</div>
</template>
```

## When to use it

- Custom gallery layouts that need a measured container width.
- SSR-sensitive layouts that should start from a seeded width.
- Responsive UIs that want stable snapped widths instead of every intermediate pixel.
