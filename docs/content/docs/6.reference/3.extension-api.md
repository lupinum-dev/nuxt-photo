title: Engine And Customization
description: The old /extend entrypoint was removed; use the root Vue API and engine package instead.
---

`@nuxt-photo/vue/extend` was removed in the engine cutover.

Use these surfaces instead:

- `@nuxt-photo/vue` for `useLightboxProvider`, primitives, and stable injection keys like `LightboxComponentKey`, `ImageAdapterKey`, and `LightboxDefaultsKey`
- `@nuxt-photo/engine` for the framework-free lightbox runtime

Practical guidance:

- Build a custom lightbox UI with `useLightboxProvider` plus Vue primitives.
- Override the default recipe lightbox globally with `LightboxComponentKey`.
- Provide a default image adapter with `ImageAdapterKey`.
- Drop to `@nuxt-photo/engine` only when you need the orchestration layer below Vue.
