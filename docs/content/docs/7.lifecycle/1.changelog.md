Use the changelog when you need to answer one question: what changed for users in this release?

Nuxt Photo keeps its release history in the root [`CHANGELOG.md`](https://github.com/lupinum-dev/nuxt-photo/blob/main/CHANGELOG.md). That file is the source of truth for:

- added, changed, fixed, removed, and deprecated behavior
- breaking changes that affect upgrades
- release-by-release user impact

## Current release shape

The active changelog follows one stable structure:

- `Unreleased` at the top while work is still landing
- grouped headings such as `Added`, `Changed`, `Removed`, and `Fixed`
- upgrade notes inline when a change affects public usage

The current `Unreleased` entry is the important one for maintainers right now. It records the engine cutover, the removal of `@nuxt-photo/vue/extend`, and the new stable customization path built around `useLightboxProvider`.

## How to use it

- Read the release entry you plan to upgrade to before updating packages.
- Check the `Unreleased` section when you are testing branch work or preparing a release.
- Follow any upgrade notes in the same entry before updating package versions.

## What belongs there

The changelog is a public record, not a commit dump. Good entries describe:

- what changed from a user or maintainer point of view
- whether the change is breaking
- what action, if any, the reader needs to take
