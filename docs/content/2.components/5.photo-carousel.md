# PhotoCarousel

`PhotoCarousel` is an inline swipeable carousel — ideal for blog posts, docs pages, and landing sections. Arrows, thumbnails, counter, captions, autoplay, and optional lightbox are all built in, each toggleable with a single prop. Powered by [Embla Carousel](https://www.embla-carousel.com/).

::carousel-playground
::

## Usage

### Basic

```vue
<script setup>
const photos = [
  { id: 1, src: '/photos/a.jpg', width: 1600, height: 1000 },
  { id: 2, src: '/photos/b.jpg', width: 1600, height: 1000 },
  { id: 3, src: '/photos/c.jpg', width: 1600, height: 1000 },
]
</script>

<template>
  <PhotoCarousel :photos="photos" />
</template>
```

Defaults: arrows on, thumbnails on, counter on, dots off, lightbox off, autoplay off.

### Toggling controls

```vue
<PhotoCarousel
  :photos="photos"
  :show-thumbnails="false"
  :show-dots="true"
  :show-counter="false"
/>
```

### Autoplay

```vue
<!-- boolean shorthand -->
<PhotoCarousel :photos="photos" :autoplay="true" />

<!-- explicit options -->
<PhotoCarousel :photos="photos" :autoplay="{ delay: 4000, stopOnInteraction: true }" />
```

When `autoplay` is truthy, PhotoCarousel prepends Embla's Autoplay plugin internally. Any user-supplied plugin named `autoplay` in the `plugins` array is filtered out (the prop wins) and a dev warning is emitted.

### With lightbox

```vue
<PhotoCarousel :photos="photos" :lightbox="true" />
```

The inline carousel becomes the thumbnail for a full lightbox. Click any slide to open it; the ghost transition animates from the on-screen slide.

### Multi-slide view

Show multiple slides in the viewport at once:

```vue
<PhotoCarousel
  :photos="photos"
  slide-size="40%"
  slide-aspect="1 / 1"
  gap="1rem"
  :show-thumbnails="false"
/>
```

### Full Embla options

Everything else Embla accepts is passed through via `options`:

```vue
<PhotoCarousel
  :photos="photos"
  :options="{ loop: true, align: 'center', slidesToScroll: 2 }"
  :thumbs-options="{ dragFree: true, containScroll: 'keepSnaps' }"
/>
```

### With CMS data

```vue
<script setup>
import type { PhotoAdapter } from '@nuxt-photo/core'

const cmsPhotos = await fetchFromCMS()

const adapter: PhotoAdapter = (item) => ({
  id: item.uuid,
  src: item.file.url,
  width: item.file.width,
  height: item.file.height,
  caption: item.title,
})
</script>

<template>
  <PhotoCarousel :photos="cmsPhotos" :photo-adapter="adapter" />
</template>
```

## Props

::field-group
  ::field{name="photos" type="PhotoItem[] | any[]" required}
  Array of photos. With `photoAdapter`, items can be any shape.
  ::

  ::field{name="photoAdapter" type="PhotoAdapter"}
  Transforms each item into a `PhotoItem`.
  ::

  ::field{name="adapter" type="ImageAdapter"}
  Custom image adapter for `src`, `srcset`, and `sizes`.
  ::

  ::field{name="options" type="EmblaOptionsType"}
  Embla options for the main carousel. Shallow-merged over internal defaults (`{ loop: false, align: 'start', containScroll: 'trimSnaps' }`).
  ::

  ::field{name="plugins" type="EmblaPluginType[]"}
  Extra Embla plugins. If `autoplay` is also set, a user-supplied plugin named `autoplay` is filtered out.
  ::

  ::field{name="thumbsOptions" type="EmblaOptionsType"}
  Embla options for the thumbnail rail. Defaults to `{ containScroll: 'keepSnaps', dragFree: true }`.
  ::

  ::field{name="showArrows" type="boolean" defaultValue="true"}
  Show prev / next arrow buttons.
  ::

  ::field{name="showThumbnails" type="boolean" defaultValue="true"}
  Show the thumbnail rail under the main carousel.
  ::

  ::field{name="showCounter" type="boolean" defaultValue="true"}
  Show the `n / total` counter overlay.
  ::

  ::field{name="showDots" type="boolean" defaultValue="false"}
  Show dot navigation below the carousel.
  ::

  ::field{name="autoplay" type="boolean | AutoplayOptionsType" defaultValue="false"}
  Enable autoplay. `true` uses defaults; pass an object for custom delay and behavior.
  ::

  ::field{name="slideSize" type="string" defaultValue='"100%"'}
  Width of each slide. Any CSS length (`'100%'`, `'40%'`, `'320px'`).
  ::

  ::field{name="slideAspect" type="string" defaultValue='"16 / 10"'}
  Slide aspect ratio. Any valid `aspect-ratio` value.
  ::

  ::field{name="gap" type="string" defaultValue='"0.75rem"'}
  Gap between slides.
  ::

  ::field{name="thumbSize" type="string" defaultValue='"5.5rem"'}
  Thumbnail height.
  ::

  ::field{name="lightbox" type="boolean | Component" defaultValue="false"}
  Enable a lightbox on slide click. `true` for default; pass a component for a custom lightbox.
  ::

  ::field{name="transition" type="LightboxTransitionOption"}
  Lightbox open/close transition mode. See [Transitions guide](/guides/transitions).
  ::

  ::field{name="slideClass" type="string"}
  Extra classes on each `.np-carousel__slide`.
  ::

  ::field{name="imgClass" type="string"}
  Extra classes on each default `<img>`.
  ::

  ::field{name="thumbClass" type="string"}
  Extra classes on each `.np-carousel__thumb`.
  ::

  ::field{name="captionClass" type="string"}
  Extra classes on the caption container.
  ::

  ::field{name="controlsClass" type="string"}
  Extra classes on the arrow controls container.
  ::
::

## Slots

Inline slots render inside the carousel itself. Lightbox slots are forwarded to the internal `PhotoGroup` (only when `lightbox !== false`).

### Inline

| Slot | Props | Description |
|---|---|---|
| `slide` | `{ photo, index, selected, open }` | Replaces the default `PhotoImage` per slide. |
| `thumb` | `{ photo, index, selected, goTo }` | Replaces the default thumbnail. |
| `caption` | `{ photo, index, count }` | Replaces the default inline caption. |
| `controls` | `{ goToPrev, goToNext, canGoToPrev, canGoToNext, selectedIndex, snapCount, goTo }` | Full navigation override. |
| `prev` | — | Content of the prev arrow button. |
| `next` | — | Content of the next arrow button. |
| `dots` | `{ snaps, selectedIndex, goTo }` | Custom dot navigation. |

### Lightbox

| Slot | Forwards to | Description |
|---|---|---|
| `lightbox-slide` | `slide` | Custom lightbox slide content. |
| `lightbox-caption` | `caption` | Custom lightbox caption. |
| `lightbox-toolbar` | `toolbar` | Custom lightbox toolbar. |

## Imperative handle

`PhotoCarousel` exposes its carousel state via template ref:

```vue
<script setup>
import { ref } from 'vue'

const carousel = ref()

function goToLast() {
  carousel.value.goTo(carousel.value.emblaApi.snapList().length - 1)
}
</script>

<template>
  <PhotoCarousel ref="carousel" :photos="photos" />
  <button @click="goToLast">Jump to last</button>
</template>
```

Exposed: `emblaApi`, `thumbsApi`, `selectedIndex`, `goTo(i, instant?)`, `goToNext(instant?)`, `goToPrev(instant?)`, `selectedSnap()`, `reInit(options?, plugins?)`.
