Photo galleries in Vue and Nuxt usually force a choice: a lightbox library that fights your layout, a layout engine without a viewer, or a CMS adapter that ships neither. **nuxt-photo is all three, layered**, so you can stay high-level with recipe components or drop to the engine and primitives when you need to.

## Try it

::photo-playground{mode="compact" :allow-thumbnail-picker="false"}
::

Start with `PhotoAlbum`. The default rows layout already includes a lightbox, keyboard navigation, gestures, and zoom — nothing to wire up.

## Why this exists

::card-group
  ::card{icon="i-lucide-layout-grid"}
  #title
  Three layout engines
  #description
  Rows (Knuth–Plass justification), columns (balanced shortest-path), and masonry — each tuned for responsive container widths.
  ::

  ::card{icon="i-lucide-image"}
  #title
  Real lightbox
  #description
  FLIP transitions from thumbnail to full-screen, swipe, pinch-to-zoom, keyboard navigation — built on the same primitives you'd reach for anyway.
  ::

  ::card{icon="i-lucide-server" to="/guides/ssr-and-performance"}
  #title
  Honest SSR story
  #description
  Rows is zero-CLS by default. Columns and masonry become zero-CLS with `defaultContainerWidth` or responsive inputs — per-breakpoint snapshots that match the hydrated layout. No CLS hand-waving.
  ::

  ::card{icon="i-lucide-layers" to="/advanced/architecture"}
  #title
  Layered, not monolithic
  #description
  Nuxt module → recipe components → Vue primitives → framework-agnostic core. Replace any layer without rewriting the rest.
  ::
::

## Architecture

::architecture-map
::

Most users stay at `@nuxt-photo/nuxt` or the recipe components. The lower layers exist so you can drop to primitives and engine APIs without changing libraries.
